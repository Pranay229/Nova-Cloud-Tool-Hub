# PWA Setup Instructions

Your project is now configured as a Progressive Web App (PWA)!

## What's Been Configured

1. **Service Worker**: Automatically generated by vite-plugin-pwa
2. **Web App Manifest**: Configured in `vite.config.ts`
3. **Offline Support**: Caching strategy for assets and fonts
4. **Install Prompt**: Users can install the app on their devices

## Icon Requirements

You need to create PNG icons for the PWA. The following sizes are required:

- `icon-192x192.png` (192x192 pixels)
- `icon-512x512.png` (512x512 pixels)

### How to Create Icons

1. **Option 1: Use the SVG icon**
   - Use `public/icon.svg` as a base
   - Convert it to PNG using an online tool like:
     - https://cloudconvert.com/svg-to-png
     - https://convertio.co/svg-png/
   - Export at 192x192 and 512x512 sizes

2. **Option 2: Design your own**
   - Create icons matching your brand
   - Use the gradient colors: #2563eb (blue) to #6366f1 (indigo)
   - Ensure icons are square and have rounded corners

3. **Option 3: Use a tool**
   - Use PWA asset generators like:
     - https://www.pwabuilder.com/imageGenerator
     - https://realfavicongenerator.net/

## Testing the PWA

1. **Build the project**:
   ```bash
   npm run build
   ```

2. **Preview the build**:
   ```bash
   npm run preview
   ```

3. **Test in browser**:
   - Open Chrome DevTools (F12)
   - Go to Application tab
   - Check "Service Workers" section
   - Check "Manifest" section
   - Test "Add to Home Screen" functionality

4. **Test offline**:
   - Open DevTools → Network tab
   - Enable "Offline" mode
   - Refresh the page - it should still work!

## Deployment

When deploying to production:
1. Ensure icons are properly sized and placed in `/public`
2. Build the project: `npm run build`
3. Deploy the `dist` folder
4. Ensure HTTPS is enabled (required for PWA)

## Browser Support

PWAs work on:
- Chrome/Edge (Desktop & Mobile)
- Firefox (Desktop & Mobile)
- Safari (iOS 11.3+)
- Samsung Internet
